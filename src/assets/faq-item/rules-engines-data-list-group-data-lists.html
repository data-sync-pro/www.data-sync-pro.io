Grouping Executable Data Lists into a Pipeline offers several key benefits:

  1. Organized UI with Variant Options
    The Pipeline Data Lists component can be placed on a Lightning page to display multiple Data Lists using different variants—including stack, tabs, and list-views—giving designers control over how data is presented and navigated.
  2. Flexible Configuration & Maintenance
    You can add, remove, reorder, or modify Executable Data Lists directly within the Pipeline—without changing the Lightning page—making updates easier to manage.
  3. Merged Actions for Shared Queries
    If multiple Data Lists in the Pipeline use the same source query, their actions are automatically merged into a single list, providing users with a unified interface.

Pipelines bring structure, flexibility, and modularity—making it easier to manage and present multiple related Data Lists in one place.
