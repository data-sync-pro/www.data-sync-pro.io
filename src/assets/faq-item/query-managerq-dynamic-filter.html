A dynamic filter in Data Sync Pro lets query filters adjust automatically at runtime based on record context, the current user, or custom Apex logic. This enables context-aware queries that adapt to data or user attributes instead of relying on static values.

How It Works:

  • Record-Based Filters: Reference fields from a record with the :FieldName syntax, including relational fields (e.g., :Account.OwnerId). In Query Builder, you can enter a static Context Record ID to test these filters. On a Lightning Record Page, the current record becomes the context automatically.

  • User-Based Filters: Use built-in tokens like :$User.Id, :$User.DirectReports, :$User.Team, or :$UserRole.Team to filter dynamically based on the logged-in user. Commonly applied to reference fields pointing to the User object.

  • Custom Apex Logic (Low-Code): For advanced scenarios, implement the DynamicFilterValue interface in an Apex class and reference it in a filter (e.g., :MyApexClass).
