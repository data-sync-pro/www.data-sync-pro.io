The Retrieve step defines how source data is queried from a connected Salesforce org. It's the entry point for record selection in both #Batches and #Data Lists.

  • In Batches: Retrieve specifies the source records for batch executionâ€”similar to the start() method in Salesforce's Database.Batchable interface.

  • In Data Lists: Retrieve defines the SOQL query used to display records to users in the UI.

At runtime, DSP automatically fetches only the fields required by downstream steps (Scoping, Match, Mapping). Fields selected in the Retrieve query are used mainly for preview and display.

This design makes query configuration simple and intuitive, while ensuring DSP executes efficiently with only the fields actually needed.
