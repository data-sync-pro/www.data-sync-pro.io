Scoping refines source data in-memory using filters and joins to define the dataset for downstream processing. It is available in all DSP rules engines, including #Batch, #Data Lists, #Triggers, #Action Buttons, and #Data Loader.

Key functions of Scoping:

  • Scope Filters (in-memory filtering): Apply DSP formulas and expressions to define which source records proceed to Match, Mapping, and Action steps.

  • Source Joins: Combine source data with another object (from any Connection) or with a JSON array. DSP supports left joins and cross joins, enabling multi-source refinement and dynamic filtering.

Note: Scoping is for advanced data shaping that cannot be expressed directly in the initial SOQL query (Retrieve). It runs after data is retrieved or uploaded, but before Match, Mapping, and Actions.
