TRIGGER_FLIPPER allows users to repeatedly trigger specific automations in DSP by checking a checkbox field on record update—or by meeting rule-driven conditions.


When used in a before insert or before update trigger, TRIGGER_FLIPPER(field_name) detects when a checkbox flips from FALSE to TRUE, records that change in memory only, and then automatically resets the checkbox back to FALSE—so it can be flipped again later.


An optional formula condition (flip_if_evaluated_true_optional) can simulate the flip automatically based on business logic—without requiring user input.


Paired with TRIGGER_IS_FLIPPED(field_name), this mechanism enables modular, reusable, and repeatable trigger-based automation—without persisting the trigger flag in the database.
