The "Use Salesforce Upsert API" field determines whether DSP should leverage Salesforce’s native Upsert API when performing upsert actions.


Key Purposes:

  1. External ID Matching – If enabled, DSP requires the Target Matching Field to be an External ID field in the target object. Salesforce will use this field to find existing records and insert new ones if no match is found.
  2. Performance Optimization – The Upsert API can improve performance by reducing separate match queries before insert/update actions.
  3. Simplified Record Matching – Instead of manually defining match logic, Salesforce handles the lookup based on the specified External ID.

Considerations:

  • Skip Record Update if No Changes cannot be used — Because the Upsert API bypasses DSP’s own Match step, it cannot verify whether the target record already holds identical values before updating.
  • External ID required — The Target Matching Field you supply must be an External ID field.
