The Permissions (Query Results) settings in a #Data List Executable specify what users are permitted to do with query results:

  1. Q: Createable? - Allows users to create new records directly from the #Data List.
  2. Q: Editable? - Enables inline and mass editing of records.
  3. Q: Editable Fields - Specifies which fields are editable when editing is enabled.
  4. Q: Deletable? - Allows users to delete single records or multiple selected records in the #Data List.
  5. Q: Downloadable? - Grants users permission to download either the full query results or only selected records (requires the custom permission “DSP: Download Data in Q”).
  6. Q: Show Row Action 'Edit'? - Enables an Edit option for each row.
  7. Q: Show Row Action 'Clone'? - Enables a Clone action for each row.
  8. Q: Show Row Action 'Delete'? - Enables a Delete option for individual records.
  9. Q: Required Custom Permissions To View - Restricts visibility of the #Data List to users with specific custom permissions.

These permission settings provide precise control over how users interact with data—ensuring secure and tailored experiences within #Data Lists.
