Actionable Data List
  • Created with a Direction-Based Executable
  • Supports cust row actions and list actions (e.g., transform, CRUD, or notify)
  • Defines both source and target logic with configurable rules
Non-Actionable Data List
  • Created with a Connection-Based Executable
  • Used primarily for data retrieval & editing
  • Still supports inline editing, but does not support custom actions


➡️ Use Direction-Based when you need interactive lists with actions.
➡️ Use Connection-Based when you only need read/edit functionality.
