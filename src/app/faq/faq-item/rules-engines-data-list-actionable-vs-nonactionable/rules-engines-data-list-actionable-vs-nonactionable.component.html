- Actionable Data List
    - Created using a Direction-based Executable
    - Supports Row Actions and List Actions (e.g., update, transform, notify)
    - Defines both source and target logic with configurable rules

- Non-Actionable Data List
    - Created using a Connection-based Executable
    - Used for data retrieval only
    - Still supports inline editing, but no custom actions

➡️ Use Direction-based for interactive lists with actions
➡️ Use Connection-based for simple, read/edit-only lists
