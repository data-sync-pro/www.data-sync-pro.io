Here’s how you can grant users access to a Trigger in DSP:

  1. Access to Executables – DSP triggers are specified in Executables. Users need Read access to these Executables.
  2. Grant "DSP: Data Sync Pro Starter" Permission Set – This provides the essential bare minimum access to DSP package objects and components for working with triggers.
  3. Automatic Field Enablement – When enabling any trigger event for the first time, DSP auto-enables "All Internal Users Have Read Access?" to create a sharing record for all internal users.
  4. Bypass Trigger Custom Permissions – Use "Bypass Triggers Custom Permissions" to assign specific bypass permissions to data migration or special users.
  5. Required Custom Permissions – Specify custom permissions in "Required Custom Permissions to Execute" so only users with those permissions can run the triggers.

These configurations ensure security by authorizing the right users, while offering flexibility to tailor access based on user personas or specific scenarios like data migration.
