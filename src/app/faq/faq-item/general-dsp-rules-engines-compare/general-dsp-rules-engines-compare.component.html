The Batch, Data List, and Trigger rules engines in DSP are Direction-based, meaning they take input from a source object, apply transformations, and perform an action on a target object. The Data Loader is similar to Batch but is Connection-based, meaning its source data comes from a CSV upload rather than a source object. The Action Button can be either Direction-based or Connection-based:

  • Connection-based Action Button – Acts as a global action with no input data.
  • Direction-based Action Button – Functions as a record action, using the current record from a Lightning record page as input.

While these rules engines share similar sub-processes, the key difference lies in how they receive input:

  • Batch – Processes all records retrieved by the Retrieve query.
  • Data List – Displays query results to users, allowing manual record selection before execution.
  • Triggers – Executes when Salesforce DML events occur, using the triggering records as input.
  • Action Button:
    • Connection-based: No input data.
    • Direction-based: Uses the current record from the Lightning record page.
  • Data Loader – Uses uploaded CSV data as input instead of querying a source object.
