Here's how you can grant users access to a #Trigger in DSP:

  1. Access to Executables: DSP #triggers are specified in Executables. Therefore, to grant users access to the #triggers, they must first have access to the Executables where the triggers are defined.
  2. Grant "DSP: Data Sync Pro Starter" Permission Set : Assign the "DSP: Data Sync Pro Starter" permission set to all internal users. This permission set provides the essential bare minimum access to the package objects and components needed to work with DSP #triggers.
  3. Automatic Field Enablement : When enabling any trigger event for the first time, DSP automatically enables the "All Internal Users Have Read Access?" field. This creates a sharing record that grants all internal users read access to the current Executable. You can disable this checkbox at any time to revoke access for all internal users.
  4. Bypass Trigger Custom Permissions : Use the "Bypass Triggers Custom Permissions" field to assign specific bypass permissions to users, such as data migration users, who are assigned one of the defined custom permissions.
  5. Required Custom Permissions : Lastly, the "Required Custom Permissions to Execute" field allows you to specify the custom permissions required for users to execute the #triggers defined in the Executable.

These configurations offer security by ensuring that only authorized users can access and execute the #triggers, while also providing flexibility to tailor access levels and permissions based on user personas or specific use cases like data migration.
