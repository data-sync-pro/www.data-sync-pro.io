Preview functions similarly to Retrieve, as both execute a SOQL query to fetch source data. However, they serve different purposes based on the rules engine in use:

  1. Retrieve is used in #Batch and #Data List to define and filter the source data during execution. It determines which records will be processed in the operation.
  2. Preview is used in #Triggers and #Action Buttons, but it does not influence the input data:
    • In #Triggers, the input data comes from the triggering context (i.e., the records being inserted, updated, or deleted).
    • In #Action Buttons, the input data comes from the loaded record in the Lightning Record Page.
  3. Both Retrieve and Preview allow users to view source records and preview transformed values before execution, helping validate the data and transformations applied.
