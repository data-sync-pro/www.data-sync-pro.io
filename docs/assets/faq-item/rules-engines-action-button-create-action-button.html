To create an #Action Button in Data Sync Pro (DSP), start by creating an Executable—either Direction-based for record actions or Connection-based for global actions. Once created, DSP provides a guided interface to configure each step of the action logic:

Guided Steps for Defining an Action Button

  1. Scoping - Refine the retrieved dataset using in-memory filters and optional joins with other datasets for multi-source enrichment.
  2. Match - Identify target records using matching rules. Supports insert-only logic, multi-field matching via transformations, and merge-specific logic.
  3. Mapping - Apply formulas and expressions to assign and transform target field values. Features auto-validation, relational field support, and a comprehensive function library.
  4. Action - Define the operations to be performed, such as Insert, Update, Upsert, Delete, Merge, Lead Convert, Send Email, and more. Configure behavior options like “Skip Record Update if No Changes” and writeback.

Additional Action on Target Settings:

  • Action on Target - Configures how the action behaves, including button label and style, visibility as list or row action (or both), confirmation prompt, and editable fields before execution.
  • Lightning Placement Options - #Action Buttons can be embedded as a Quick Action, a standalone Executable Action Button, or grouped using Pipeline Action Buttons.

This structured, intuitive interface ensures your #Action Buttons are modular, reusable, and fully aligned with your business logic—without writing any code.
