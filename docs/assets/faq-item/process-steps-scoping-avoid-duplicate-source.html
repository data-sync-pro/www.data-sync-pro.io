To avoid duplicate source records based on specific conditions, use the IS_FIRST_IN_ITERATION(...expressions) function in a Scope Filter. It evaluates whether a particular combination of field values appears for the first time during the current execution cycle and excludes any duplicates that follow.

Example Usage

To process only one record per unique combination of FirstName, LastName, and Email:
IS_FIRST_IN_ITERATION(FirstName, LastName, Email)

Note

This filter works within each batch or CSV upload, not across the entire job. For better control, sort the source data so that records with the same key fields appear together. Use IS_FIRST_IN_ITERATION to efficiently handle deduplicationâ€”without pre-processing or complex logic.
